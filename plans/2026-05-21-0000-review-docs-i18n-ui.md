### [High] hreflang タグの欠落 — SEO 多言語対応不足（対応済み）
- **指摘者**: i18n Expert
- **場所**: `docs/index.html`, `docs/ja/index.html` (head セクション)
- **影響**: 検索エンジンが英語版と日本語版の関連性を認識できず、重複コンテンツ判定リスク・SEO 分散の可能性
- **対応**: 各ページの `<head>` に `<link rel="alternate" hreflang="en/ja">` + canonical + og:locale タグを追加。現在の実装:
  ```html
  <!-- docs/index.html -->
  <link rel="canonical" href="https://armaniacs.github.io/sasurahime/" />
  <link rel="alternate" hreflang="en" href="https://armaniacs.github.io/sasurahime/" />
  <link rel="alternate" hreflang="ja" href="https://armaniacs.github.io/sasurahime/ja/" />
  <meta property="og:locale" content="en_US" />

  <!-- docs/ja/index.html -->
  <link rel="canonical" href="https://armaniacs.github.io/sasurahime/ja/" />
  <link rel="alternate" hreflang="en" href="https://armaniacs.github.io/sasurahime/" />
  <link rel="alternate" hreflang="ja" href="https://armaniacs.github.io/sasurahime/ja/" />
  <meta property="og:locale" content="ja_JP" />
  ```

### [High] ナビゲーション — 言語切り替え時のドキュメントリンク動線が不明確（対応済み）
- **指摘者**: UI/UX Expert
- **場所**: `docs/index.html`:209, `docs/ja/index.html`:185（当時）
- **影響**: 言語切り替え時にドキュメントリンクも連動して遷移するかが不明確。UX 一貫性が低い
- **対応**: 以下のナビゲーション統合を実施:
  - Docページの `switchLang()` JS を拡張し、JA表示時に nav ラベルを「使い方」「ターゲット一覧」に切り替え（`_layouts/doc.html`）
  - 言語スイッチャーUIを全ページで `.lang-btn` ボタンスタイルに統一（`index.html`, `ja/index.html`）
  - フッターを拡充: 全3ドキュメントへのリンク（使い方、対応ターゲット一覧、ターゲット追加方法）を両言語版に追加
  - Docページフッターに「Top」リンクを追加（`_layouts/doc.html`）
  - HOWTO-USE.md の両言語セクション末尾に相互参照リンクを追加

### [Medium] canonical タグの欠落（対応済み）
- **指摘者**: i18n Expert
- **場所**: `docs/index.html`, `docs/ja/index.html`
- **影響**: 言語別 URL の優先順位を検索エンジンに明示できず、インデックス戦略が不明確
- **対応**: 各ページに `<link rel="canonical">` を追加（hreflang と同時対応）

### [Medium] CTA ボタンのリンク先・テキストが不明確（対応済み）
- **指摘者**: UI/UX Expert
- **場所**: `docs/index.html`:227-230（当時）
- **影響**: 「Read the docs」が Markdown ファイルへの相対リンクで、実際の表示形式が不明。ユーザーの期待と遷移先がずれるリスク
- **対応**: `Read the docs` → `View Documentation`、`ドキュメントを読む` → `ドキュメントを見る` に変更。リンク先は拡張子なしの Jekyll permalink で問題なし

### [Medium] 日本語版ランディングページのセクションラベルが英語のまま（対応済み）
- **指摘者**: UI/UX Expert
- **場所**: `docs/ja/index.html`:215 および他セクション（当時）
- **影響**: 「Features」「Supported Targets」「How It Works」等のセクションラベルが英語のまま残っており、デザイン一貫性が低下
- **対応**: 全ラベルを日本語化: `Features` → `機能`、`Supported Targets` → `対応ターゲット`、`How It Works` → `仕組み`、`Installation` → `インストール`、`Name` → `名前の由来`

### [Medium] ドキュメントの言語対応の非対称性（対応済み）
- **指摘者**: i18n Expert
- **場所**: `docs/HOWTO-USE.md`, `docs/SUPPORTED.md`, `docs/HOWTO-ADD-target.md`
- **影響**: 日本語版ランディングページから英語ドキュメントへのアクセスパスが不明確
- **対応**: 
  - 全3ドキュメントに日本語 `<details>` セクションを追加済み（bilingual 形式）
  - フッターに全3ドキュメントへのリンクを追加（両言語版）
  - HOWTO-USE.md に EN/JA 両方の相互参照リンクを追加

## 確認済みの良好点（更新）

- **言語切り替えナビゲーション**: 全ページで `.lang-btn` ボタンスタイルに統一。DocページはJS言語認識でラベル自動切替
- **HTML lang 属性**: 英語 `lang="en"`・日本語 `lang="ja"` が正しく設定
- **SEO メタタグ**: hreflang / canonical / og:locale が全ランディングページに実装済み
- **構造的対称性**: hero / features / targets / workflow / installation セクションが両言語で対称
- **共有CSS**: インライン重複なし、`docs/assets/style.css` に一元化
- **レスポンシブグリッド**: `minmax(250px, 1fr)` でモバイル対応
- **ダークテーマ**: GitHub Dark 準拠で視認性良好
- **相互リンク**: 全3ドキュメントへの導線がフッター・本文から確保
- **Markdown の多言語サポート**: `<details>` タグで両言語対応済み
