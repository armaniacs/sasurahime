## はじめに

Rust 製 macOS キャッシュクリーナー **sasurahime** の v0.1.27 をリリースしました。

このバージョンで、設定ファイルによるカスタマイズ、Xcode キャッシュの部分削除、削除履歴の記録といった主要機能が揃い、ひと区切りを迎えました。

## v0.1.27 の新機能

### 設定のカスタマイズがさらに充実

`~/.config/sasurahime/config.toml` に以下の設定が書けるようになりました：

```toml
# スキャンから除外するクリーナー
exclude = ["huggingface", "ollama"]

# 自分でキャッシュディレクトリを追加
[[custom]]
name = "my-project"
path = "~/work/.cache"

# クリーナーごとのフィルタ
[cleaner.act]
older_than_days = 30

[cleaner.colima]
larger_than_mb = 500
```

- **`exclude`**：指定したクリーナーを `scan` や TUI の一覧から消せます。
- **`[[custom]]`**：sasurahime が標準で対応していないツールのキャッシュも、パスを書くだけで管理できます。
- **`[cleaner.<name>]`**：`older_than_days`（N 日より古いものだけ削除）や `larger_than_mb`（N MB 以上のものだけ削除）をクリーナーごとに設定できます。

`--config <path>` フラグで、デフォルトの `~/.config/sasurahime/config.toml` とは別の設定ファイルを読み込むことも可能です。

### Xcode キャッシュの部分削除

Xcode の DerivedData と Archives を個別に選んで削除できるようになりました。

```
  xcode > DerivedData  (15.3 GB)
  xcode > Archives     (5.2 GB)
```

TUI で Xcode を選ぶとサブカテゴリが展開され、**DerivedData だけ**、**Archives だけ**、あるいは**両方**を選べます。

コマンドラインからも指定できます：

```bash
sasurahime clean xcode --sub derived-data
sasurahime clean xcode --sub derived-data,archives
```

何も指定しなければ従来通り DerivedData のみ削除されるため、既存のスクリプトはそのまま動きます。

### 削除履歴と統計

`sasurahime stats` コマンドが追加されました：

```bash
$ sasurahime stats
Total freed:  12.5 GB
Runs:         15

Recent cleanups:
  #  Date                Cleaner        Size
  1  2026-05-27 10:30   uv             500.0 MB
  2  2026-05-26 22:15   brew           1.2 GB
```

履歴は `clean` のたびに `~/.local/share/sasurahime/history.json` に自動追記されます。`--dry-run` のときは記録されません。

```bash
# 直近 5 件だけ表示
sasurahime stats --last 5
```

履歴ファイルがない場合や破損している場合も、エラーメッセージを表示して正常終了します（クラッシュしません）。

## 品質改善

### 安全性の強化

コードレビューで指摘された以下のセキュリティ強化を実施しました：

- **環境変数による任意パス削除の防止**：`TF_PLUGIN_CACHE_DIR` や `PUB_CACHE` 環境変数が `/etc` などのシステムパスを指している場合、警告を出してデフォルトパスにフォールバックするようになりました。`is_safe_delete_target()` 関数がシステムクリティカルなパスをブロックします。
- **ダウンロード全削除防止**：`sasurahime clean downloads` にデフォルトの 30 日フィルタを追加。設定ファイルの `[cleaner.downloads].older_than_days` で調整できます。
- **カスタムパスの安全性チェック**：設定ファイルの `[[custom]]` で指定したパスがシステムディレクトリを指していないかを検証し、危険なパスは拒否します。

### テストカバレッジの拡充

- TUI の選択ロジックを純関数として分離し、6 つの単体テストを追加
- APFS Snapshot クリーナーの E2E テストを新規作成
- history.json のスキーマバージョン管理を追加（後方互換性確保）

### `--permanent` のヘルプ文言を修正

`--help` に表示される `--permanent` フラグの説明文を修正しました。

**修正前:**
```
--permanent        Move deleted files to Trash instead of permanent removal (default: enabled)
```

「Permanent」というフラグ名なのに「Trash に移動」と書いてあり、さらに「(default: enabled)」がどちらの動作を指すのか不明瞭でした。

**修正後:**
```
--permanent        Permanently delete files instead of moving to Trash. By default, files are moved to macOS Trash for safety
```

「Permanent = 完全削除」というフラグ名と説明が一致し、デフォルトが Trash（安全側）であることが明確になりました。

### 新しい Rust コンパイラに対応

Rust 1.95.0 でのビルドを通すため、コードを一部修正しました：

- テストモジュールの配置をファイル末尾に移動（`src/cleaner.rs`）
- 不要な参照演算子を削除（`tests/config.rs`）

## テスト状況

**442 tests, 0 failures**（288 unit + 154 integration/E2E、24 テストファイル）。

全クリーナーが `detect()` + `clean()` + `dry_run` のテストを持ち、主要な機能も十分にカバーされています。

## まとめ

v0.1.0 からの 10 日間で、キャッシュクリーナーとしての基本機能は一通り揃いました：

- **40 以上のクリーンターゲット**（言語ランタイム、パッケージマネージャ、ブラウザ、IDE、Docker 周辺、Xcode…）
- **scan / clean / explore / stats** のサブコマンド
- **インタラクティブ TUI**（dialoguer の MultiSelect）
- **--yes モード**（cron/CI 向け非インタラクティブ一括削除）
- **設定ファイル**（exclude / custom target / per-cleaner フィルタ）
- **削除履歴と統計**
- **並列スキャン**（rayon による高速化）
- **ゴミ箱モード**（デフォルトで Trash へ移動、Finder から復元可能）
