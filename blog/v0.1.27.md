## はじめに

Rust 製 macOS キャッシュクリーナー **sasurahime** の v0.1.27 をリリースしました。

このバージョンで、PBI-A から PBI-G までの全 7 つのプロダクトバックログアイテム（合計 22 ストーリーポイント）が完了し、一区切りを迎えました。

前回の v0.1.26（PBI-D/G）からの追加要素を中心に、v0.1.27 で何が変わったのかを紹介します。

## v0.1.27 の新機能

### 設定のカスタマイズがさらに充実（PBI-E）

`~/.config/sasurahime/config.toml` に以下の設定が書けるようになりました：

```toml
# スキャンから除外するクリーナー
exclude = ["huggingface", "ollama"]

# 自分でキャッシュディレクトリを追加
[[custom]]
name = "my-project"
path = "~/work/.cache"

# クリーナーごとのフィルタ
[cleaner.act]
older_than_days = 30

[cleaner.colima]
larger_than_mb = 500
```

- **`exclude`**：指定したクリーナーを `scan` や TUI の一覧から消せます。
- **`[[custom]]`**：sasurahime が標準で対応していないツールのキャッシュも、パスを書くだけで管理できます。
- **`[cleaner.<name>]`**：`older_than_days`（N 日より古いものだけ削除）や `larger_than_mb`（N MB 以上のものだけ削除）をクリーナーごとに設定できます。

`--config <path>` フラグで、デフォルトの `~/.config/sasurahime/config.toml` とは別の設定ファイルを読み込むことも可能です。

### Xcode キャッシュの部分削除（PBI-D）

Xcode の DerivedData と Archives を個別に選んで削除できるようになりました。

```
  xcode > DerivedData  (15.3 GB)
  xcode > Archives     (5.2 GB)
```

TUI で Xcode を選ぶとサブカテゴリが展開され、**DerivedData だけ**、**Archives だけ**、あるいは**両方**を選べます。

コマンドラインからも指定できます：

```bash
sasurahime clean xcode --sub derived-data
sasurahime clean xcode --sub derived-data,archives
```

何も指定しなければ従来通り DerivedData のみ削除されるため、既存のスクリプトはそのまま動きます。

### 削除履歴と統計（PBI-G）

`sasurahime stats` コマンドが追加されました：

```bash
$ sasurahime stats
Total freed:  12.5 GB
Runs:         15

Recent cleanups:
  #  Date                Cleaner        Size
  1  2026-05-27 10:30   uv             500.0 MB
  2  2026-05-26 22:15   brew           1.2 GB
```

履歴は `clean` のたびに `~/.local/share/sasurahime/history.json` に自動追記されます。`--dry-run` のときは記録されません。

```bash
# 直近 5 件だけ表示
sasurahime stats --last 5
```

履歴ファイルがない場合や破損している場合も、エラーメッセージを表示して正常終了します（クラッシュしません）。

## 品質改善

### 22 名のコードレビュー（Checking Team）

今回のマージ前に、多角的コードレビューシステム **Checking Team**（22 名の専門家エージェント）によるレビューを実施しました。

主なフィードバックと対応：

- **🔴 任意パス削除リスク**（Red Team）→ `is_safe_delete_target()` を実装。`TF_PLUGIN_CACHE_DIR` や `PUB_CACHE` 環境変数が `/etc` などを指している場合、警告を出してデフォルトパスにフォールバック。
- **🔴 ダウンロード全削除リスク**（Edge & Mobile）→ `sasurahime clean downloads` にデフォルト 30 日フィルタを追加。
- **🔴 history.json スキーマ進化**（SRE/Ops）→ `HistoryEntry` に `version` フィールドを追加。`#[serde(default)]` で旧形式とも互換。
- **🔴 テスト不足**（Test Experts）→ `is_safe_delete_target` のテスト追加、カスタムパス拒否テスト、`APFS Snapshot` E2E テスト追加。

レビューのスコアは **82.4/100（Rank A）**。レポートは `plans/` 以下に保存されています。

### カバレッジギャップ監査

全 24 テストファイルを対象にカバレッジ監査を実施。**スコア 8.5/10**。重大なギャップは見つかりませんでした。

不足していた部分にはテストを追加：

- `interactive.rs` の TUI 選択ロジックを純関数に抽出し、6 つの単体テストを追加
- `tests/apfs_snapshot.rs` を新規作成（CLI エントリポイントの動作確認）

## テスト状況

**442 tests, 0 failures**（288 unit + 154 integration/E2E、24 テストファイル）。

全クリーナーが `detect()` + `clean()` + `dry_run` のテストを持ち、設定・履歴・並列スキャンなどの横断的機能も十分にカバーされています。

## これまでとこれから

v0.1.0（2026-05-18）からの 10 日間で、キャッシュクリーナーとしての基本機能は一通り揃いました：

- **42 のクリーンターゲット**（言語ランタイム、パッケージマネージャ、ブラウザ、IDE、Docker 周辺、Xcode…）
- **scan / clean / explore / stats** のサブコマンド
- **インタラクティブ TUI**（dialoguer の MultiSelect）
- **--yes モード**（cron/CI 向け非インタラクティブ一括削除）
- **設定ファイル**（exclude / custom target / per-cleaner フィルタ）
- **削除履歴と統計**
- **並列スキャン**（rayon による高速化）
- **堅牢なエラーハンドリング**（権限エラー・ファイルロックをスキップ）
- **ゴミ箱モード**（デフォルトで Trash へ移動、Finder から復元可能）
