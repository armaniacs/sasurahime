# PBI-008: インタラクティブ選択モード

## ユーザーストーリー
**sasurahime を初めて使う開発者**として、スキャン結果を見ながら削除したい項目をチェックボックスで選んで実行したい、なぜなら `clean` サブコマンドを個別に覚えなくても直感的に操作できるから。

## ビジネス価値
- CLI に不慣れなユーザーでも安全に使える
- スキャン→選択→削除を1コマンド `sasurahime` で完結させる

## BDD 受け入れシナリオ

```gherkin
Scenario: インタラクティブモードで複数項目を選択して削除する
  Given sasurahime scan が uv / brew / bun のキャッシュを検出している
  When `sasurahime` を引数なしで実行する
  Then スキャン結果がカテゴリ別に一覧表示される
  And チェックボックスで削除したい項目を選択できる
  And 選択確定後に合計回収予定サイズが表示され確認を求められる
  And "y" で削除が実行され回収サイズが表示される

Scenario: 何も選択せずに終了する
  When `sasurahime` を実行してすべての選択を外した状態で確定する
  Then "Nothing selected. Exiting." と表示して終了する
  And 何も削除されない

Scenario: --yes フラグで確認をスキップする
  When `sasurahime --yes` を実行する
  Then 全項目が自動選択されて確認プロンプトなしで削除が実行される
```

## 受け入れ基準
- [ ] 引数なし実行でインタラクティブモードが起動する
- [ ] カーソルキーとスペースキーでチェックボックスを操作できる
- [ ] 選択後に合計回収予定サイズを表示して確認を求める
- [ ] `--yes` フラグで全自動実行（CI / スクリプト用途）
- [ ] TTY 非接続時（パイプ等）はインタラクティブモードを起動せずエラーメッセージを表示

## t_wada スタイル テスト戦略
```
E2Eテスト:
- --yes フラグ付きで実行し、モッククリーナーが全て呼ばれることを確認

統合テスト:

単体テスト:

```

## 実装アプローチ
- **Outside-In**: --yes フラグ付き E2E テストから開始（TUI なしで自動テスト可能）
- TUI: `dialoguer` クレートの `MultiSelect` を使用
- PBI-001〜007 の全クリーナーが `Cleaner` trait を実装している前提
